# Monopoly

Графическая реализация настольной игры **Monopoly** на Python с использованием **CustomTkinter** для интерфейса и **Pygame** для музыкального сопровождения.

## О проекте

Этот проект — desktop-игра по мотивам классической Monopoly. Игроки по очереди бросают кубики, перемещаются по игровому полю, покупают компании, получают ренту, продают активы, строят улучшения и выбывают из игры при банкротстве.

В приложении предусмотрены:

- стартовый экран с выбором количества игроков;
- экран презентации перед началом партии;
- основное игровое поле с интерактивными кнопками;
- окно победителя;
- музыкальное сопровождение и визуальные ассеты.

## Возможности

- графический интерфейс на **CustomTkinter**;
- поддержка от **2 до 6 игроков**;
- движение по полю с помощью броска двух кубиков;
- покупка компаний и станций;
- начисление ренты;
- клетки **Chance**, **Jail**, **Start**, **Free Park**;
- строительство домов и отелей;
- продажа собственности;
- определение победителя после выбывания остальных игроков;
- отдельные изображения и музыкальные треки для игровых сцен.

```text
Monopoly/
├── main.py
```

## Установка и запуск

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/yvi7693/Monopoly.git
cd Monopoly
```

### 2. Создайте и активируйте виртуальное окружение

**Windows**

```bash
python -m venv .venv
.venv\Scripts\activate
```

**macOS / Linux**

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установите зависимости

```bash
pip install -r requirements.txt
```

### 4. Запустите игру

```bash
python main.py
```

## Как играть

1. Запустите приложение.
2. На стартовом экране выберите количество игроков.

![Стартовый экран](images/StartScreen.png)

3. Нажмите **Start**.
4. В игровом окне используйте кнопку **MOVE**, чтобы выполнить ход текущего игрока.

![Игровой экран](images/GameScreen2.png)

5. Если игрок попадает на свободную собственность, игра предложит купить её.
6. При необходимости используйте действия **Sell** и **Build** для управления активами.
7. Игра продолжается до тех пор, пока не останется один игрок — он и становится победителем.

![Экран победителя](images/FinishScreen.png)

## Архитектура

Проект разделён на несколько уровней:

- **models** — хранит сущности игры: поле, клетки, кубики, игроков и типы собственности;
- **controllers** — реализует правила игры, банковские операции, покупку/продажу и управление очередью игроков;
- **view** — отвечает за графический интерфейс;
- **presenter** — координирует взаимодействие между интерфейсом и игровой логикой.

Такая структура упрощает поддержку проекта и делает код понятнее при расширении функциональности.
